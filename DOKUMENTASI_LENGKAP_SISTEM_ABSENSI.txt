1. PENGANTAR SISTEM
================================================================================

Sistem Absensi Archemi adalah aplikasi web yang dibuat untuk mengelola kehadiran 
karyawan di perusahaan. Sistem ini dibuat menggunakan framework Laravel 10 dengan 
template engine Blade untuk tampilan yang modern dan responsif.

TUJUAN SISTEM:
- Memudahkan pencatatan kehadiran karyawan
- Mengelola data karyawan dan departemen
- Mengatur pengajuan cuti dan izin
- Memberikan laporan kehadiran yang akurat
- Meningkatkan efisiensi administrasi HR

MENGAPA MEMILIH LARAVEL?
Laravel dipilih karena:
- Framework PHP yang populer dan mudah dipelajari
- Memiliki fitur keamanan yang baik
- Dokumentasi lengkap dan komunitas besar
- Cocok untuk aplikasi skala menengah hingga besar
- Mendukung pengembangan yang cepat (rapid development)

3. STRUKTUR DATABASE
================================================================================

Database dirancang dengan 5 tabel utama yang saling berhubungan:

A. TABEL USERS (Pengguna Sistem)
   Kolom:
   - id: ID unik pengguna
   - name: Nama pengguna
   - email: Email untuk login
   - password: Password terenkripsi
   - role: Peran (admin/hr/employee)
   - employee_id: Referensi ke tabel employees
   - is_active: Status aktif/nonaktif
   
   FUNGSI: Menyimpan data login dan hak akses pengguna

B. TABEL DEPARTMENTS (Departemen)
   Kolom:
   - id: ID unik departemen
   - name: Nama departemen
   - description: Deskripsi departemen
   - head_of_department: Kepala departemen
   - is_active: Status aktif/nonaktif
   
   FUNGSI: Mengelompokkan karyawan berdasarkan divisi kerja

C. TABEL EMPLOYEES (Karyawan)
   Kolom:
   - id: ID unik karyawan
   - employee_id: Nomor induk karyawan
   - name: Nama lengkap
   - email: Email karyawan
   - phone: Nomor telepon
   - address: Alamat
   - birth_date: Tanggal lahir
   - gender: Jenis kelamin
   - position: Jabatan
   - department_id: ID departemen (foreign key)
   - hire_date: Tanggal bergabung
   - salary: Gaji
   - status: Status karyawan (active/inactive/terminated)
   - profile_photo: Foto profil
   
   FUNGSI: Menyimpan data lengkap karyawan

D. TABEL ATTENDANCES (Absensi)
   Kolom:
   - id: ID unik absensi
   - employee_id: ID karyawan (foreign key)
   - date: Tanggal absensi
   - clock_in: Jam masuk
   - clock_out: Jam keluar
   - break_start: Mulai istirahat
   - break_end: Selesai istirahat
   - total_hours: Total jam kerja
   - status: Status kehadiran (present/absent/late/sick/permission)
   - notes: Catatan
   - location: Lokasi absensi
   
   FUNGSI: Mencatat kehadiran harian karyawan

E. TABEL LEAVES (Cuti/Izin)
   Kolom:
   - id: ID unik pengajuan
   - employee_id: ID karyawan (foreign key)
   - type: Jenis cuti (annual/sick/maternity/paternity/emergency/unpaid)
   - start_date: Tanggal mulai
   - end_date: Tanggal selesai
   - total_days: Total hari
   - reason: Alasan cuti
   - status: Status (pending/approved/rejected)
   - admin_notes: Catatan admin
   - approved_at: Waktu persetujuan
   - approved_by: ID yang menyetujui
   - attachment: File lampiran
   
   FUNGSI: Mengelola pengajuan cuti dan izin

RELASI ANTAR TABEL:
- Users ↔ Employees (One-to-One)
- Departments → Employees (One-to-Many)
- Employees → Attendances (One-to-Many)
- Employees → Leaves (One-to-Many)
- Users → Leaves (One-to-Many, untuk approval)

4. FITUR-FITUR UTAMA
================================================================================

A. SISTEM AUTHENTICATION (Login/Logout)
   FITUR:
   - Login dengan email dan password
   - Logout yang aman
   - Session management
   - Password hashing dengan bcrypt
   
   CARA KERJA:
   - Pengguna memasukkan email dan password
   - Sistem memverifikasi dengan database
   - Jika valid, session dibuat dan pengguna diarahkan ke dashboard
   - Password disimpan dalam bentuk hash untuk keamanan

B. ROLE-BASED ACCESS CONTROL (Kontrol Akses Berdasarkan Peran)
   PERAN PENGGUNA:
   1. ADMIN: Akses penuh ke semua fitur
   2. HR: Akses ke manajemen karyawan dan persetujuan cuti
   3. EMPLOYEE: Akses terbatas untuk absensi dan pengajuan cuti pribadi
   
   CARA KERJA:
   - Setiap pengguna memiliki role yang tersimpan di database
   - Middleware Laravel mengecek role sebelum mengakses halaman
   - Menu dan tombol disesuaikan dengan hak akses pengguna

C. DASHBOARD INTERAKTIF
   FITUR:
   - Statistik real-time (total karyawan, kehadiran hari ini, cuti pending)
   - Quick actions (clock in/out, tambah karyawan, input absensi)
   - Recent activities (aktivitas terbaru)
   - Pengajuan cuti yang perlu disetujui
   
   CARA KERJA:
   - Controller mengambil data dari database
   - Data dihitung dan diformat
   - Ditampilkan dalam bentuk cards dan grafik
   - Auto-refresh untuk data real-time

D. MANAJEMEN KARYAWAN
   FITUR:
   - CRUD (Create, Read, Update, Delete) data karyawan
   - Upload foto profil
   - Filter dan pencarian
   - Pagination untuk data banyak
   
   CARA KERJA:
   - Form input dengan validasi
   - File upload untuk foto profil
   - Data disimpan ke database dengan relasi ke departemen
   - Otomatis membuat akun user untuk karyawan baru

E. MANAJEMEN DEPARTEMEN
   FITUR:
   - CRUD departemen
   - Statistik jumlah karyawan per departemen
   - Status aktif/nonaktif
   
   CARA KERJA:
   - Form sederhana untuk input departemen
   - Validasi nama departemen harus unik
   - Pengecekan sebelum hapus (tidak bisa hapus jika masih ada karyawan)

F. SISTEM ABSENSI
   FITUR:
   - Clock in/out real-time dengan AJAX
   - Input absensi manual oleh admin/HR
   - Perhitungan jam kerja otomatis
   - Deteksi keterlambatan
   - Multiple status kehadiran
   
   CARA KERJA CLOCK IN/OUT:
   1. Karyawan klik tombol "Clock In"
   2. JavaScript mengirim request AJAX ke server
   3. Server mencatat waktu saat ini
   4. Sistem mengecek apakah terlambat (setelah jam 08:00)
   5. Data disimpan ke database
   6. Response dikembalikan ke browser
   7. Halaman diupdate tanpa refresh
   
   CARA KERJA INPUT MANUAL:
   1. Admin/HR mengisi form absensi
   2. Sistem validasi data input
   3. Perhitungan total jam kerja otomatis
   4. Data disimpan dengan status sesuai pilihan

G. MANAJEMEN CUTI DAN IZIN
   FITUR:
   - Pengajuan cuti online
   - Multiple jenis cuti
   - Upload dokumen pendukung
   - Sistem approval workflow
   - Tracking status pengajuan
   
   CARA KERJA PENGAJUAN:
   1. Karyawan mengisi form pengajuan cuti
   2. Sistem menghitung durasi hari otomatis
   3. File attachment diupload (opsional)
   4. Status awal "pending"
   5. Notifikasi ke admin/HR
   
   CARA KERJA APPROVAL:
   1. Admin/HR melihat daftar pengajuan pending
   2. Klik tombol approve/reject
   3. Modal popup untuk input catatan
   4. Status diupdate di database
   5. Timestamp dan ID approver dicatat

5. CARA KERJA SETIAP FITUR DETAIL
================================================================================

A. PROSES LOGIN
   LANGKAH TEKNIS:
   1. User mengakses halaman login (/login)
   2. Memasukkan email dan password
   3. Laravel Auth middleware memverifikasi
   4. Password di-hash dan dibandingkan dengan database
   5. Jika cocok, session dibuat
   6. User diarahkan ke dashboard sesuai role

B. PROSES CLOCK IN
   LANGKAH TEKNIS:
   1. User klik tombol "Clock In" di dashboard
   2. JavaScript function clockIn() dipanggil
   3. AJAX POST request ke route /attendance/clock-in
   4. AttendanceController->clockIn() method dijalankan
   5. Sistem cek apakah sudah clock in hari ini
   6. Ambil waktu saat ini dengan Carbon::now()
   7. Cek keterlambatan (> 08:00)
   8. Simpan ke database dengan status yang sesuai
   9. Return JSON response ke browser
   10. JavaScript update tampilan

C. PROSES PENGAJUAN CUTI
   LANGKAH TEKNIS:
   1. User mengisi form di /leaves/create
   2. JavaScript menghitung durasi hari otomatis
   3. Form di-submit ke LeaveController->store()
   4. Validasi input (tanggal, jenis cuti, alasan)
   5. Upload file attachment jika ada
   6. Hitung total_days dengan Carbon
   7. Simpan ke database dengan status "pending"
   8. Redirect ke index dengan success message

D. PROSES APPROVAL CUTI
   LANGKAH TEKNIS:
   1. Admin/HR buka halaman /leaves
   2. Klik tombol approve pada pengajuan
   3. Modal popup terbuka dengan form catatan
   4. Submit ke route /leaves/{id}/approve
   5. LeaveController->approve() method dijalankan
   6. Update status menjadi "approved"
   7. Catat waktu approval dan ID approver
   8. Redirect dengan success message

E. PROSES UPLOAD FOTO
   LANGKAH TEKNIS:
   1. User pilih file di form employee
   2. Validasi file (type, size) di controller
   3. Laravel Storage::disk('public')->store() menyimpan file
   4. Path file disimpan di database
   5. File dapat diakses via Storage::url()

6. KEAMANAN SISTEM
================================================================================

A. AUTHENTICATION & AUTHORIZATION
   - Password hashing dengan bcrypt
   - Session-based authentication
   - CSRF protection pada semua form
   - Role-based access control dengan custom middleware
   - Middleware untuk proteksi route berdasarkan role
   - Custom RoleMiddleware untuk granular access control

B. ROLE-BASED ACCESS CONTROL
   - ADMIN: Full access ke semua fitur sistem
   - HR: Access ke employee management, department, attendance manual, leave approval
   - EMPLOYEE: Access terbatas ke personal attendance dan leave application
   - Middleware protection pada controller level dan route level

C. VALIDASI INPUT
   - Server-side validation untuk semua form
   - Sanitasi input untuk mencegah XSS
   - File upload validation (type, size)
   - Database constraint untuk data integrity
   - Real-time validation dengan JavaScript

D. KEAMANAN DATABASE
   - Eloquent ORM untuk mencegah SQL injection
   - Foreign key constraints
   - Soft deletes untuk data penting
   - Encrypted sensitive data
   - Unique constraints untuk data critical

E. KEAMANAN FILE
   - Upload file ke storage/public
   - Validasi file type dan size
   - Symbolic link untuk akses file
   - Automatic cleanup saat delete record

F. ERROR HANDLING & SECURITY
   - Custom error pages (403, 404, 500)
   - Graceful error handling
   - Unauthorized access protection
   - Session timeout handling
